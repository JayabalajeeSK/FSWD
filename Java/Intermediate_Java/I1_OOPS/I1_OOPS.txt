‚úÖ 1. CLASS & OBJECT
Class

A class is a blueprint or template.
It defines variables + methods but does not occupy memory.

Object

An object is a real instance of a class created using new.
It occupies memory.

‚úî Example
class Student {
    String name;
    int age;

    void display() {
        System.out.println(name + " - " + age);
    }
}

public class Main {
    public static void main(String[] args) {
        Student s = new Student();  // object created
        s.name = "JB";
        s.age = 22;
        s.display();
    }
}

‚úÖ Constructors

Special methods used to initialize objects.

Default Constructor

Provided by Java automatically if you don‚Äôt create one.

Parameterized Constructor

Constructor with arguments.

‚úî Example: Constructor Overloading
class Car {
    String model;
    int price;

    // Default constructor
    Car() {
        model = "Unknown";
        price = 0;
    }

    // Parameterized constructor
    Car(String model, int price) {
        this.model = model;
        this.price = price;
    }

    void show() {
        System.out.println(model + " - " + price);
    }
}

‚úÖ 2. ENCAPSULATION

Wrapping data + methods together using private variables and public getters/setters.

‚úî Example

    private int balance;

    public void setBalance(int amount) {
        balance = amount;
    }

    public int getBalance() {
        return balance;
    }

| Modifier  | Access               |
| --------- | -------------------- |
| private   | same class only      |
| default   | same package         |
| protected | package + subclasses |
| public    | everywhere           |

‚úî Types

Used to call parent constructor or parent method.

‚≠ê Method Overriding (Runtime Polymorphism)

Child class provides a new definition for parent method.

class Phone {
    void show() { System.out.println("Phone"); }
}

class SmartPhone extends Phone {
    @Override
    void show() { System.out.println("SmartPhone"); }
}

‚≠ê Dynamic Method Dispatch

Reference is parent type, object is child type.

Phone p = new SmartPhone();
p.show();  // SmartPhone

‚≠ê Compile-time Polymorphism ‚Äî Overloading

Same method name, different parameters.

class Add {
    int sum(int a, int b) { return a + b; }
    int sum(int a, int b, int c) { return a + b + c; }
}

‚≠ê Interfaces

Supports:

Static methods (Java 8)

        System.out.println("Bark");
    }
}

‚≠ê Association vs Aggregation vs Composition
‚úî Association

General relationship.

class Teacher {}
class Student {}

‚úî Aggregation (weak)

Child can survive without parent.

    Student s;
}

‚úî Composition (strong)

Child cannot survive without parent.

class House {
    private Room r = new Room(); // tightly bound
}

üî• INTERVIEW PROGRAM ‚Äî FULL OOP IMPLEMENTATION

A single demo program using ALL important OOP concepts.

// INTERFACE
interface Payment {
    double calculate(double amount);

    default void success() {
        System.out.println("Payment Successful!");
    }
}

// ABSTRACT CLASS
abstract class User {
    String name;
    User(String name) { this.name = name; }

    abstract void greet();
}

// CLASS WITH ENCAPSULATION
class Wallet {
    private double balance;

    public void addMoney(double amount) { balance += amount; }
    public double getBalance() { return balance; }
}

// INHERITANCE + OVERRIDING + SUPER
class Customer extends User implements Payment {
    Wallet wallet = new Wallet();
    
    Customer(String name) {
        super(name);
    }

    @Override
    void greet() {
        System.out.println("Hello " + name);
    }

    @Override
    public double calculate(double amount) {
        return amount * 0.95; // 5% discount
    }
}

// MAIN PROGRAM
public class OOPDemo {
    public static void main(String[] args) {

        User u = new Customer("JB"); // Dynamic dispatch
        u.greet();

        Customer c = (Customer) u;
        c.wallet.addMoney(1000);

        double finalAmt = c.calculate(200);
        System.out.println("Final Amount: " + finalAmt);

        c.success();
        System.out.println("Wallet Balance: " + c.wallet.getBalance());
    }
}

5. What is HAS-A Relationship?

Association ‚Üí Aggregation ‚Üí Composition.

üí† Association ‚Üí Aggregation ‚Üí Composition

These are OOP relationships between classes (how two classes are connected).

All three represent a HAS-A relationship, but the strength of the relationship increases in this order:

‚úî Association (weakest)
‚úî Aggregation (medium)
‚úî Composition (strongest)

Let‚Äôs explain with real-life examples + Java code.

üü¶ 1. ASSOCIATION (Weak Relationship)

Two classes are related, but both can exist independently.

üëâ Example:
A Teacher teaches a Student, but

Teacher can exist without Student

Student can exist without Teacher

‚úî Code:
class Teacher {
    String name;
}

class Student {
    String name;
}

// They interact, but neither owns the other.


üìå Key point:
No ownership. They just use each other.

üü© 2. AGGREGATION (Weak HAS-A / ‚ÄúWhole‚ÄìPart‚Äù relationship)

One class contains another class,
but child can survive even if parent is destroyed.

üëâ Example:
A College HAS students,
but a Student can exist even if the College closes.

‚úî Code:
class Student {
    String name;
}

class College {
    Student student; // HAS-A (Aggregation)
}


üìå Key point:

College has students

Students don‚Äôt depend on College

Independent life cycle

üü• 3. COMPOSITION (Strongest HAS-A)

Child cannot exist without parent.
Parent owns and controls the child object completely.

üëâ Example:
A House has Rooms,
but a Room cannot exist without the House.

‚úî Code:
class Room {
    // part of house
}

class House {
    private Room room = new Room(); // Composition
}


üìå Key point:

Strong ownership

If House is destroyed ‚Üí Rooms also destroyed

Same life cycle

| Relationship    | Meaning        | Dependency | Example            | Life Cycle  |
| --------------- | -------------- | ---------- | ------------------ | ----------- |
| **Association** | Connection     | None       | Teacher ‚Üî Student  | Separate    |
| **Aggregation** | HAS-A (weak)   | Partial    | College ‚Üí Students | Independent |
| **Composition** | HAS-A (strong) | Full       | House ‚Üí Room       | Dependent   |
